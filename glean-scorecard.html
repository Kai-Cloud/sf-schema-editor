Salesforce CRM Connector Gap Analysis: Glean vs Microsoft

Data-driven comparison based on documented connector capabilities. April 2026. Schema Editor ·

Closure strategy:
• OOB: MVP Fix the 5 core objects (Account, Contact, Lead, Opportunity, Case) with correct schema in the OOB connector.
• OOB: Post-MVP Additional objects (Tasks, Events, Case Comments, Files, Campaigns, Products, Orders, CPQ, Territory, Custom Objects, etc.) planned for the OOB connector after MVP ships.
• Hybrid mode Combines OOB index with real-time API queries (FCC). OOB provides the crawled index; FCC supplements with live SOQL for date, numeric, aggregation, and real-time data scenarios. Architectural equivalent of Glean's Live Mode + sync-ed connector.
• Platform Requires platform-level infrastructure (e.g. hourly ACL full crawl). Neither OOB nor custom connector can implement without platform support.

Gap Assessment: Path to Parity

The gap to Glean is significant but not insurmountable. Most gaps fall into well-scoped engineering work rather than fundamental platform limitations.

OOB: MVP (schema fix for 5 core objects): estimated 4-6 weeks. This is the highest-priority work and addresses the most visible quality issues.
OOB: Post-MVP (Tasks, Events, Case Comments, Files, Campaigns, Products, Orders, CPQ, Territory, Custom Objects): estimated 2-4 weeks per object. Incremental additions to the existing crawl pipeline.
Hybrid mode (OOB index + FCC real-time SOQL): combines crawled index with query-time API calls for date, numeric, aggregation, and live data scenarios that pure index cannot serve. Architectural equivalent of Glean's Live Mode + sync-ed connector.
Platform (scalability and response quality): timeline TBD. Handling 18+ objects with hundreds of properties in a single connector requires platform investment in schema scalability and CRM-domain response quality. Hourly ACL full crawl is a lower priority (no customer escalations yet). These gaps cannot be closed by the connector team alone.

Overall: With focused execution, the OOB MVP + Post-MVP work (8-14 weeks) closes the most customer-visible gaps. Hybrid mode extends coverage to date/numeric/aggregation patterns and real-time data fetch. Platform investment is the long pole for full parity.
